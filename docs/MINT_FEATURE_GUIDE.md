# EasyMint 功能指引 — 给 Mint 看的产品手册

> 当用户问「这个怎么用」「按钮在哪」「怎么操作」时，从本文档查找答案。
> 本文档按「用户想做什么」组织，而非按代码结构组织。

---

## 1. 整体布局

EasyMint 是一个四栏桌面应用，从你（Mint）的视角看：

```
┌──────────┬──────────────┬────────────────────────┬──────────────┐
│ 侧边图标栏 │  左面板       │  中间编辑区               │  右面板       │
│  (44px)  │  (220px)     │  (Tab 栏 + 内容区)        │  (280px)     │
│          │  文件树 或    │  Chat 面板 或 Editor     │  项目进度     │
│  + 新建   │  会话列表     │                         │  鱼骨图       │
│  📁 文件  │             │                         │  任务列表     │
│  💬 会话  │             │                         │  万能按钮     │
│  ⚙ 设置   │             │                         │             │
└──────────┴──────────────┴────────────────────────┴──────────────┘
```

- 左面板和右面板都可以**拖拽调整宽度**（拖中间的分隔线）
- 左面板和右面板都可以**折叠/展开**（点面板标题栏的箭头按钮，折叠后在边缘会出现恢复按钮）

---

## 2. 页面路由

| 路由 | 页面 | 何时出现 |
|------|------|---------|
| 首次启动 | OnboardingPage | 首次打开 App，配置 API Key 和模型列表 |
| `/` | ProjectPage（无项目） | 日常启动，未选中任何项目，使用 workspace 目录 |
| `/project/:projectId` | ProjectPage（有项目） | 选中了某个项目 |

> **重要**：`/` 路由下，工作目录是 `~/EasyMintProject/workspace/`，这是**临时工作区**，不应在此创建项目。应引导用户点「新建项目」。

## 3. 侧边图标栏（最左侧 44px 竖条）

从上到下排列的按钮：

| 图标 | 名称 | 功能 |
|------|------|------|
| **+** | 新建 | 点击弹出下拉菜单，包含 5 个选项 |
| 📁 | 项目文件 | 点击切换到文件树模式，左面板显示项目文件 |
| 💬 | 会话 | 点击切换到会话列表模式，左面板显示对话记录 |
| 🖥 | 终端 | 灰色禁用态，tooltip 显示「终端 · 即将推出」 |
| ⚙ | 设置 | 底部，打开设置弹窗 |

### + 新建下拉菜单

点击 + 按钮弹出：

1. **新建项目** — 打开 NewProjectDialog（5 步表单）
2. **打开项目** — 打开「打开项目」弹窗，列出已有项目
3. ─── 分隔线 ───
4. **新建文件** — 弹出输入框，输入文件名（在当前项目下创建）
5. **新建文件夹** — 弹出输入框，输入文件夹名（在当前项目下创建）
6. ─── 分隔线 ───
7. **新建窗口** — 打开一个新的 EasyMint 窗口

> 注意：新建文件/文件夹的功能目前只打印到 console，尚未完全实现。

## 4. 左面板 — 项目文件（文件树模式）

### 标题栏
- 标题文字：**项目文件**
- **⊟ 折叠全部**按钮 — 收起所有展开的文件夹

### 文件树
- 文件夹前有 **▶ 箭头**，点击展开/折叠子目录
- 文件点击后在中间区域打开 Editor Tab
- 被修改过的文件后面有绿色 **M** 标记
- 当前选中的文件高亮（淡绿背景）

### 空状态
- 如果没有项目打开，显示「暂无项目文件」

## 5. 左面板 — 会话列表（会话模式）

### 顶部
- **+ 新建会话** 按钮（虚线边框），点击在中间区域打开一个新的 Chat Tab

### 会话列表
- 按时间分组：「置顶」「今天」「昨天」「更早」
- 每个会话项显示：
  - 绿色圆点（未置顶）/ 黄色圆点（已置顶）
  - 会话标题
  - 最后活跃时间
- 点击某个会话 → 在中间区域打开 Chat Tab，加载该会话的历史记录
- 当前活跃的会话有淡绿背景高亮

### 右键菜单
在会话项上右键弹出：
- **置顶/取消置顶** — 将重要会话固定在列表顶部
- **重命名** — 内联编辑会话标题，Enter 确认，Esc 取消
- **删除** — 删除该会话及其所有历史记录（不可恢复）

## 6. 中间编辑区 — Tab 栏 + 内容

### Tab 栏
- 每个 Tab 有一个圆点图标（文件=灰色 / 聊天=绿色）+ 标题 + 关闭 ✕
- 当前活跃的 Tab 白底高亮
- 文件 Tab 如果未保存，标题前有脏标记
- 可同时打开多个文件 Tab 和多个聊天 Tab

### Tab 类型

#### 文件 Tab → EditorPanel（Monaco 编辑器）
- 支持语言高亮：JS/TS/JSON/HTML/CSS/Markdown
- 主题：浅绿 Mint 配色
- **Ctrl+S** 保存文件
- 保存后右下角短暂显示「已保存」
- 右侧显示行号（灰色，当前行号绿色）
- 语法高亮颜色：注释灰色、关键字紫色、字符串绿色、数字橙色、类型蓝色、函数红色

#### 聊天 Tab → ChatPanel（对话面板）
见第 7 节。

### 欢迎页（无 Tab 时）
- 中间显示：「欢迎使用 EasyMint」
- 「点击按钮或者与 Mint 聊天开始创建项目」

## 7. ChatPanel — 聊天面板（核心功能）

### 7.1 消息区域

- **用户消息气泡**：右对齐，绿色背景（accent 色），圆角
- **AI 消息气泡**：左对齐，白色背景 + 边框，圆角
- 每条消息显示时间戳

### 7.2 附件功能

用户可以上传文件附着在消息上：

- **图片上传**：
  - 点击工具栏的图片图标
  - 或直接 Ctrl+V 粘贴剪贴板中的图片
  - 或拖拽图片到输入框
  - 支持 PNG/JPEG/GIF/WebP/BMP/SVG
  - 图片预览在输入框上方，点击可放大查看（lightbox）
  - 上传后生成缩略图 dataUrl，直接嵌入消息

- **文档上传**：
  - 点击工具栏的文档图标
  - 支持 PDF/DOC/DOCX/MD/TXT/CSV/XLSX/代码文件等
  - 按文件类型显示彩色图标：
    - 🔴 红色 → 文档类（md, txt, pdf, doc 等）
    - 🟢 绿色 → 代码文件（ts, js, py, rs, go 等）
    - 🔵 蓝色 → 数据文件（csv, xls, xlsx 等）
    - 🟣 紫色 → 配置文件（json, yaml, toml, env 等）
    - ⚪ 灰色 → 其他文件

- 附件在用户消息气泡中展示，发送前可以逐个删除

### 7.3 AI 消息中的工具调用展示

AI 消息里除了文本回复，还展示工具调用过程（每个步骤默认显示一种块）：

- **🤔 思考过程**（thinking）— AI 的推理内容
- **🔧 工具调用**（tool_use）— 如 Read/Bash/Write/Edit，显示工具名和参数
- **📋 工具结果**（tool_result）— 工具执行返回的内容

### 7.4 智能快捷按钮

在特定的 AI 回复下方会自动出现快捷操作按钮：

| 触发条件 | 按钮文案 | 点击效果 |
|---------|---------|---------|
| AI 回复中包含「帮我初始化开发环境」 | 「帮我初始化开发环境 →」 | 自动发送这条消息 |
| AI 回复中包含「开始分配开发任务」且项目处于「任务规划」阶段 | 「开始分配开发任务 →」 | 自动发送任务分配指令 |

### 7.5 输入区域

- **输入框**：多行 textarea（3 行可见），支持 Shift+Enter 换行，Enter 直接发送
- **发送按钮**（纸飞机图标）：输入为空时灰色禁用，有内容时绿色可点击
- **停止按钮**（红色方块）：AI 正在回复时替换发送按钮，点击立即中断 AI

### 7.6 底部工具栏

从左到右：

| 控件 | 说明 |
|------|------|
| 🖼 上传图片按钮 | 打开文件选择器，选图片 |
| 📄 上传文档按钮 | 打开文件选择器，选文档 |
| 权限模式下拉 | 选择 AI 执行操作时的权限策略 |
| 权限标签 | 显示「权限」文字 |
| 模型选择下拉 | 切换当前会话使用的 AI 模型 |
| 余额显示 | 显示 API 余额，点击刷新 |

### 7.7 权限模式说明

| 选项 | 含义 |
|------|------|
| **智能判断** | AI 自动判断是否需要用户确认 |
| **只读** | AI 只能读取文件，不能写入或执行命令 |
| **手动确认** | 编辑文件自动过，执行命令需要用户确认 |
| **完全自主** | 跳过所有权限检查，AI 可以自由操作 |

### 7.8 AI 思考中状态

AI 处理消息时，输入框上方显示：
- 三个绿色跳动圆点动画
- 状态文字（如「思考中...」「处理中...」）

### 7.9 空状态

无对话时显示：「开始对话，让 Mint 帮你开发项目。」

## 8. TaskPanel — 右面板（项目进度）

### 8.1 标题栏
- 标题：**项目进度**
- **> 收起面板**按钮 — 折叠右面板

### 8.2 鱼骨图进度指示器

显示项目所处的 6 个阶段及其完成状态：

```
需求采集 ─→ 技术选型 ─→ 环境初始化 ─→ 任务规划 ─→ 正在开发 ─→ 开发完成
```

- **绿色实心圆** = 已完成阶段
- **绿色脉冲圆** = 当前阶段（显示阶段名称标签）
- **灰色空心圆** = 未开始阶段
- 鼠标悬停某个阶段节点时高亮显示
- 连接线颜色：有当前阶段时绿色，否则灰色

阶段检测逻辑（自动从文件系统推导）：

| 阶段 | 判定条件 |
|------|---------|
| 需求采集 | `需求规格.md` 不存在 |
| 技术选型 | `需求规格.md` 存在但 `init.sh` 不存在 |
| 环境初始化 | `init.sh` 存在但未执行完成 |
| 任务规划 | 环境就绪但 `task.json` 为空 |
| 正在开发 | `task.json` 中有未完成的任务 |
| 开发完成 | 所有任务 `passes: true` |

### 8.3 任务列表

- 标题行显示：**开发任务** + 「N/M 完成」
- 每个任务项有一行：
  - 状态图标：
    - 🟢 绿色勾 → 已完成（done）
    - 🟡 黄色脉冲圆 → 执行中（running）
    - 🔴 红色叉 → 失败（failed）
    - ⚪ 灰色空心圆 → 待开始（pending）
  - 任务标题
- 鼠标悬停任务项时展开显示任务描述
- 执行中的任务行有淡绿背景
- 列表自动滚动到当前执行中的任务

### 8.4 万能推进按钮（叶子按钮）

- 面板底部，大按钮（叶子图标）
- 下方文字：「如果你不知道要做什么，就点一下」
- 点击效果：
  1. 如果已有 Mint 聊天 Tab → 切换到该 Tab → 发送 `CONTINUE_NEXT_STEP` 指令
  2. 如果没有聊天 Tab → 新建一个聊天 Tab → 发送指令

`CONTINUE_NEXT_STEP` 指令内容：检查项目当前阶段和进度，总结当前状态，然后继续推进下一步工作。

## 9. NewProjectDialog — 新建项目弹窗

### 流程

5 步自适应表单，顶部有步骤点指示器（当前步骤的圆点最长）：

| 步骤 | 名称 | 内容 | 自适应规则 |
|------|------|------|-----------|
| 1 | 项目概述 | 名称*、描述、项目目录*、目标用户、项目形式（可多选+添加）、完成度 | 全部显示 |
| 2 | 功能清单 | 手动添加功能项 + 「Mint 推荐」按钮（AI 自动分析项目信息推荐功能） | 全部显示 |
| 3 | 视觉风格 | 自定义风格输入 + 14 种预设风格下拉选择 | CLI 项目跳过 |
| 4 | 技术选型 | 成本预算、技术偏好输入 + 「Mint 推荐」按钮、可展开的技术参考选项 | 全部显示 |
| 5 | 部署方式 | 本地部署 vs 云端部署 | 全部显示 |

### 关键行为

- **项目名称如果是中文**：第 1 步点击「下一步」时，自动用临时 workspace 会话翻译为英文目录名（小写、连字符分隔）
- **Mint 推荐功能**：调用 AI 根据项目信息推荐功能清单，返回结果解析为功能项填入表单
- **Mint 推荐技术**：调用 AI 推荐技术方案，填入技术偏好文本框
- **创建项目**：最后一步点击「创建项目」，发送 `PROJECT_INIT_INSTRUCTION` 给 AI，然后跳转到项目页面
- **取消项目**：取消时会删除已创建的项目目录

## 11. SettingsDialog — 设置弹窗

5 个 Tab：

### 11.1 通用 Tab

| 设置项 | 说明 |
|--------|------|
| **主题** | 固定为「亮色 Mint（仅亮色）」，无暗色切换 |
| **默认项目路径** | 新建项目时的默认父目录，workspace 也在此路径下。默认 `~/EasyMintProject` |
| **API Base URL** | API 服务地址，默认 `https://api.deepseek.com/anthropic` |
| **API Key** | API 密钥，可切换显示/隐藏 |
| **模型列表** | 可手动添加/删除模型名，或「从 API 获取」自动拉取 |
| **默认模型** | 新会话的默认模型（从模型列表中选或手动输入） |
| **开发选项** | 三个开关（见下表） |
| **上传缓存** | 管理已上传的临时文件，按时间/大小排序，支持全选/清理 |
| **Token 消耗参考** | 展示不同模式下的预估 Token 消耗（1-4 星） |

开发选项开关：

| 开关 | 说明 |
|------|------|
| 评估模式 | 每轮开发后自动触发 Evaluator 验证（Playwright 自动化测试） |
| TDD 模式 | 先编写测试代码再实现功能 |
| 截图验证 | 每轮完成后截图并通过视觉模型分析 |

### 11.2 提示词 Tab

- 可自定义系统提示词（当前实现细节在 `PromptSettings` 组件中）

### 11.3 Agent 模板 Tab

- 列出已创建的 Agent 模板（名称 + 类型标签）
- 点击「编辑」→ 表单填入模板详情
- 点击「删除」→ 删除模板
- 新建/编辑表单字段：
  - 名称、类型（Mint/Orchestrator/Builder/Evaluator）
  - 描述（SDK 用于匹配调用时机）
  - 系统提示词
  - 工具列表（逗号分隔）
  - 模型（可选，不填继承默认）

### 11.4 Skill Tab

- 列出全局和项目级的 Skill
- 每个 Skill 显示名称和描述
- 可启用/禁用（toggle 开关）
- 可隐藏（不删除文件，写入 hiddenSkills 配置）
- 「导入 Skill」按钮 — 选择文件夹导入新 Skill
- 与 Claude Code 共用 `~/.claude/skills/` 目录

### 11.5 MCP Tab

- 列出已配置的 MCP 服务器（名称 + 类型标签 + 连接信息）
- 每个服务器可启用/禁用
- API Keys 管理区域：列出所有 MCP 服务器需要的密钥
  - 输入框（密码模式，可切换显示）
  - 密钥自动注入到对应 MCP 服务器的环境变量
  - 已填 key 显示绿色，未填显示黄色警告
- 与 Claude Code 共享 `.claude.json` 配置

## 12. OnboardingPage — 首次启动设置

仅在首次运行 EasyMint 时出现（通过 `localStorage` 的 `easymint_setup_complete` 标记控制）。

3 个步骤：

| 步骤 | 名称 | 内容 |
|------|------|------|
| 1 | 配置 API | 填写 API Base URL 和 API Key |
| 2 | 获取模型 | 点击「获取模型列表」从 API 拉取可用模型，也可「跳过，稍后设置」 |
| 3 | 准备就绪 | ✓ 图标 + 确认信息 + 「进入工作台」按钮 |

完成设置后写入 `setupComplete` 到主进程，后续启动直接进入工作台。

## 13. 项目生命周期（用户完整操作流程）

```
1. 新建项目
   → 打开 NewProjectDialog，填写 5 步表单
   → 项目创建成功，自动跳转

2. 需求采集（Mint 自动进行）
   → Mint 分析表单数据 + 对话
   → 输出 docs/需求规格.md
   → 鱼骨图阶段：需求采集 ✓ → 技术选型

3. 技术选型（Mint 自动进行）
   → 输出 docs/架构设计.md + README.md + CLAUDE.md
   → 生成 init.sh
   → 鱼骨图阶段：技术选型 ✓ → 环境初始化

4. 环境初始化（Mint 自动执行）
   → 执行 bash init.sh
   → 鱼骨图阶段：环境初始化 ✓ → 任务规划

5. 任务规划
   → 点击 AI 回复下方的「开始分配开发任务 →」按钮
   → Mint 将功能拆解为 task.json
   → 鱼骨图阶段：任务规划 ✓ → 正在开发

6. 开发执行
   → 点击 TaskPanel 叶子按钮
   → 或与 Mint 对话推动开发
   → Mint/Orchestrator 调度 Builder/Evaluator 执行任务
   → 鱼骨图阶段：正在开发 → 开发完成

7. 持续迭代
   → 在 Chat 面板提新需求
   → Mint 追加到 task.json → 继续开发
```

## 14. 常见用户问题标准回答

### Q: 怎么新建项目？
**A:** 点击左侧最上面的 **+** 按钮，选择「新建项目」。会弹出一个 5 步表单：填写项目名称和类型 → 功能清单 → 视觉风格 → 技术选型 → 部署方式。填完点击「创建项目」，我会自动帮你初始化项目。

### Q: 怎么打开已有项目？
**A:** 点击左侧 **+** 按钮，选择「打开项目」，会列出所有你创建过的项目，点击即可打开。如果你已经在一个项目里，可以从左侧 **+** 菜单里选择「新建窗口」，在新窗口中打开另一个项目。

### Q: 怎么看项目文件？
**A:** 点击左侧的 📁 **项目文件**按钮，左面板会显示项目的文件树。点击文件夹展开，点击文件会在中间区域打开编辑器。

### Q: 怎么看历史对话？
**A:** 点击左侧的 💬 **会话**按钮，左面板会显示所有对话记录。点击任意一条可以在中间区域打开继续查看。

### Q: 怎么切换 AI 模型？
**A:** 在聊天输入框下方的工具栏里，有一个模型选择下拉框，点击切换即可。可用的模型列表在设置 → 通用 → 模型列表中管理。

### Q: 怎么控制 AI 的权限？
**A:** 在聊天工具栏里有一个权限模式下拉框：
- 智能判断：让 AI 自己判断
- 只读：AI 只能看不能改
- 手动确认：重要操作需要你点头
- 完全自主：AI 自由操作，适合信任的场景

### Q: 那个叶子按钮是干嘛的？
**A:** 那是「万能推进按钮」，在右面板最下面。当你不知道下一步该做什么的时候，点一下它，我会自动检查项目进度并推进下一步。它相当于在聊天里说「继续推进」。

### Q: 怎么上传图片或文件给 AI 看？
**A:** 三种方式：
1. 点击聊天输入框下方工具栏的 🖼 图片图标或 📄 文档图标
2. 直接 Ctrl+V 粘贴剪贴板里的图片
3. 把文件拖到输入框里

### Q: 怎么停止 AI 的回复？
**A:** 当 AI 正在回复时，输入框右下角的发送按钮会变成红色方块停止按钮，点击即可中断。

### Q: 怎么管理 AI 对话记录？
**A:** 在左面板的会话列表中，右键点击会话可以：
- 重命名 — 给会话起个好记的名字
- 置顶 — 把重要的会话固定在列表最上面
- 删除 — 删掉不需要的会话

### Q: 怎么配置 Agent 模板？
**A:** 打开设置（左侧底部 ⚙ 图标）→ Agent 模板 Tab。可以创建不同类型的 Agent（Builder/Evaluator/Orchestrator），自定义它们的系统提示词和可用工具。

### Q: 怎么管理 Skill？
**A:** 打开设置 → Skill Tab。可以看到全局和项目级的 Skill，可以开启/关闭/隐藏。「导入 Skill」按钮可以从本地文件夹导入新的 Skill。Skill 和 Claude Code 共用 `~/.claude/skills/` 目录。

### Q: 怎么配置 MCP 服务器？
**A:** 打开设置 → MCP Tab。MCP 服务器和 Claude Code 共享配置，用 `claude mcp add` 命令添加。API Keys 在同一个 Tab 里管理。

### Q: 面板太窄/太宽了怎么办？
**A:** 把鼠标移到左面板或右面板的边缘，会出现一个可以拖拽的分隔线，左右拖动调整宽度。也可以点击面板标题栏上的箭头按钮折叠/展开面板。

### Q: 怎么改项目默认存放路径？
**A:** 打开设置 → 通用 → 默认项目路径，修改后新建项目时默认会使用新路径。

### Q: 怎么删项目？
**A:** 在「打开项目」弹窗中，鼠标悬停在项目上，右侧会出现 ✕ 按钮，点击删除。注意：这只是从 EasyMint 的记录中删除，不会删掉你电脑上实际的项目文件。

## 15. Mint 行为约束（需要遵守的规则）

以下规则来自系统提示词和产品设计，当用户提出相关要求时请据此回答：

1. **workspace 下不建项目**：如果当前工作目录是 `workspace/`，提醒用户点击「新建项目」按钮。不要直接在 workspace 目录下创建项目文件。

2. **多任务（2+）必须写入 task.json**：判断工作量，如果需求拆解后有 2 个或以上任务，写入 task.json 并提示用户使用任务面板，不要自己逐个实现。

3. **极简项目可以直接做**：如果项目极其简单（单文件 HTML、不到 50 行、不需要依赖），直接一步到位，告诉用户「项目已完成」。

4. **表单数据是参考不是指令**：用户填的表单代表他当前的认知，你应该像 PM 一样审视，有矛盾或更好的方案主动指出。

5. **需求拆解要结构化**：从用户目标出发 → 描述工作流 → 树状编号 → 标注输入/输出/异常 → 覆盖 CRUD → 输出原子任务清单。
