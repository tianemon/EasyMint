# EasyMint 产品功能指引

> 当用户问「这个怎么用」「按钮在哪」「怎么操作」时，从这里查找答案。
> 按「用户想做什么」组织。

## 1. 整体布局

EasyMint 是一个四栏桌面应用：

```
┌──────────┬──────────────┬────────────────────────┬──────────────┐
│ 侧边图标栏 │  左面板       │  中间编辑区               │  右面板       │
│  (44px)  │  (220px)     │  (Tab 栏 + 内容区)        │  (280px)     │
│          │  文件树 或    │  Chat 面板 或 Editor     │  项目进度     │
│  + 新建   │  会话列表     │                         │  鱼骨图       │
│  📁 文件  │             │                         │  任务列表     │
│  💬 会话  │             │                         │  万能按钮     │
│  ⚙ 设置   │             │                         │             │
└──────────┴──────────────┴────────────────────────┴──────────────┘
```

- 左面板和右面板都可以**拖拽调整宽度**（拖中间的分隔线）
- 左面板和右面板都可以**折叠/展开**（点面板标题栏的箭头按钮，折叠后在边缘会出现恢复按钮）

## 2. 页面路由

| 路由 | 页面 | 何时出现 |
|------|------|---------|
| 首次启动 | OnboardingPage | 首次打开 App，配置 API Key 和模型列表 |
| `/` | ProjectPage（无项目） | 日常启动，未选中任何项目，使用 workspace 目录 |
| `/project/:projectId` | ProjectPage（有项目） | 选中了某个项目 |

> **重要**：`/` 路由下，工作目录是 `~/EasyMintProject/workspace/`，这是**临时工作区**，不应在此创建项目。应引导用户点「新建项目」。

## 3. 侧边图标栏（最左侧 44px 竖条）

| 图标 | 名称 | 功能 |
|------|------|------|
| **+** | 新建 | 点击弹出下拉菜单，包含 5 个选项 |
| 📁 | 项目文件 | 点击切换到文件树模式，左面板显示项目文件 |
| 💬 | 会话 | 点击切换到会话列表模式，左面板显示对话记录 |
| 🖥 | 终端 | 灰色禁用态，tooltip 显示「终端 · 即将推出」 |
| ⚙ | 设置 | 底部，打开设置弹窗 |

### + 新建下拉菜单

1. **新建项目** — 打开 NewProjectDialog（5 步表单）
2. **打开项目** — 打开「打开项目」弹窗，列出已有项目
3. ─── 分隔线 ───
4. **新建文件** — 弹出输入框，输入文件名（在当前项目下创建）
5. **新建文件夹** — 弹出输入框，输入文件夹名（在当前项目下创建）
6. ─── 分隔线 ───
7. **新建窗口** — 打开一个新的 EasyMint 窗口

> 注意：新建文件/文件夹的功能目前只打印到 console，尚未完全实现。

## 4. 左面板 — 项目文件（文件树模式）

- 标题文字：**项目文件**
- **⊟ 折叠全部**按钮 — 收起所有展开的文件夹
- **< 收起面板**按钮 — 折叠整个左面板
- 文件夹前有 **▶ 箭头**，点击展开/折叠子目录
- 文件点击后在中间区域打开 Editor Tab
- 被修改过的文件后面有绿色 **M** 标记
- 当前选中的文件高亮（淡绿背景）

## 5. 左面板 — 会话列表（会话模式）

- **+ 新建会话** 按钮（虚线边框），点击在中间区域打开一个新的 Chat Tab
- 会话按时间分组：「置顶」「今天」「昨天」「更早」
- 每个会话项显示：绿色/黄色圆点 + 标题 + 最后活跃时间
- 点击某个会话 → 在中间区域打开 Chat Tab，加载历史记录
- 当前活跃的会话有淡绿背景高亮

### 右键菜单
- **置顶/取消置顶** — 将重要会话固定在列表顶部（黄色圆点标记）
- **重命名** — 内联编辑会话标题，Enter 确认，Esc 取消
- **删除** — 删除该会话及其所有历史记录（不可恢复）

## 6. 中间编辑区 — Tab 栏 + 内容

- 每个 Tab：圆点图标（文件=灰色 / 聊天=绿色）+ 标题 + 关闭 ✕
- 当前活跃的 Tab 白底高亮
- 文件 Tab 如果未保存，标题前有脏标记
- 可同时打开多个文件 Tab 和多个聊天 Tab

### 文件 Tab → Monaco 编辑器
- 支持语言高亮：JS/TS/JSON/HTML/CSS/Markdown
- 主题：浅绿 Mint 配色
- **Ctrl+S** 保存文件，保存后右下角显示「已保存」
- 行号（灰色，当前行绿色）

### 欢迎页（无 Tab 时）
中间显示：「欢迎使用 EasyMint」「点击按钮或者与 Mint 聊天开始创建项目」

## 7. ChatPanel — 聊天面板

### 消息
- 用户消息气泡：右对齐，绿色背景（accent 色），圆角
- AI 消息气泡：左对齐，白色背景 + 边框，圆角
- 每条消息显示时间戳

### 附件功能
- **图片上传**：点击图片图标 / Ctrl+V 粘贴 / 拖拽到输入框
  支持 PNG/JPEG/GIF/WebP/BMP/SVG，点击图片可放大查看
- **文档上传**：点击文档图标
  支持 PDF/DOC/DOCX/MD/TXT/CSV/XLSX/代码文件等
  按文件类型显示彩色图标：
  - 🔴 红色 → 文档类（md, txt, pdf, doc）
  - 🟢 绿色 → 代码文件（ts, js, py, rs, go）
  - 🔵 蓝色 → 数据文件（csv, xls, xlsx）
  - 🟣 紫色 → 配置文件（json, yaml, toml, env）
  - ⚪ 灰色 → 其他文件

### AI 消息中的工具调用展示
- 🤔 思考过程（thinking）
- 🔧 工具调用（tool_use）— 显示工具名和参数
- 📋 工具结果（tool_result）

### 智能快捷按钮
| 触发条件 | 按钮文案 | 点击效果 |
|---------|---------|---------|
| AI 回复含「帮我初始化开发环境」 | 「帮我初始化开发环境 →」 | 自动发送 |
| AI 回复含「开始分配开发任务」且处于规划阶段 | 「开始分配开发任务 →」 | 自动发送 |

### 输入操作
- 多行 textarea，Shift+Enter 换行，Enter 直接发送
- 发送按钮（纸飞机图标）：有内容时绿色可点击
- 停止按钮（红色方块）：AI 回复中替换发送按钮，点击立即中断

### 底部工具栏
| 控件 | 说明 |
|------|------|
| 🖼 上传图片按钮 | 打开文件选择器 |
| 📄 上传文档按钮 | 打开文件选择器 |
| 权限模式下拉 | 智能判断 / 只读 / 手动确认 / 完全自主 |
| 模型选择下拉 | 切换当前会话使用的 AI 模型 |
| 余额显示 | API 余额，点击刷新 |

### 权限模式说明
| 选项 | 含义 |
|------|------|
| **智能判断** | AI 自动判断是否需要用户确认 |
| **只读** | AI 只能读取文件，不能写入或执行命令 |
| **手动确认** | 编辑文件自动过，执行命令需要用户确认 |
| **完全自主** | 跳过所有权限检查，AI 可以自由操作 |

### 状态
- AI 处理消息时显示三个绿色跳动圆点 + 状态文字（如「思考中...」）
- 无对话时显示：「开始对话，让 Mint 帮你开发项目。」

## 8. TaskPanel — 右面板（项目进度）

### 鱼骨图进度指示器
6 个阶段，自动从文件系统检测：

```
需求采集 ─→ 技术选型 ─→ 环境初始化 ─→ 任务规划 ─→ 正在开发 ─→ 开发完成
```

- **绿色实心圆** = 已完成 | **绿色脉冲圆** = 当前阶段 | **灰色空心圆** = 未开始
- 鼠标悬停高亮，当前阶段显示标签

阶段检测逻辑：

| 阶段 | 判定条件 |
|------|---------|
| 需求采集 | `需求规格.md` 不存在 |
| 技术选型 | `需求规格.md` 存在但 `init.sh` 不存在 |
| 环境初始化 | `init.sh` 存在但未执行完成 |
| 任务规划 | 环境就绪但 `task.json` 为空 |
| 正在开发 | `task.json` 中有未完成的任务 |
| 开发完成 | 所有任务 `passes: true` |

### 任务列表
- 标题：「开发任务」+ 「N/M 完成」
- 状态图标：🟢done / 🟡running / 🔴failed / ⚪pending
- 鼠标悬停展开任务描述
- 自动滚动到当前执行中的任务

### 万能推进按钮（叶子按钮）
- 面板底部，叶子图标大按钮
- 下方文字：「如果你不知道要做什么，就点一下」
- 点击效果：如果已有聊天 Tab → 切到该 Tab → 发送「检查项目进度，继续推进下一步」；没有则新建聊天 Tab 后发送

## 9. NewProjectDialog — 新建项目弹窗

5 步自适应表单：

| 步骤 | 名称 | 自适应规则 |
|------|------|-----------|
| 1 | 项目概述 | 全部显示。名称*、描述、目录*、目标用户、项目形式（多选）、完成度 |
| 2 | 功能清单 | 全部显示。手动添加 + 「Mint 推荐」AI 按钮 |
| 3 | 视觉风格 | CLI 项目跳过。14 种预设 + 自定义输入 |
| 4 | 技术选型 | 全部显示。成本预算 + 技术偏好 + 「Mint 推荐」+ 可展开参考 |
| 5 | 部署方式 | 全部显示。本地部署 vs 云端部署 |

关键行为：
- 中文项目名自动翻译为英文目录名（小写、连字符分隔）
- 「Mint 推荐」调用 AI 推荐功能清单/技术方案
- 取消项目时会删除已创建的项目目录

## 10. 打开项目弹窗

- 列出所有已保存的项目（名称、路径、最后打开日期）
- 点击跳转，悬停出现 ✕ 删除按钮（仅删除记录，不删除实际文件）
- 目录已删除的项目显示红色警告「（目录已删除）」，点击无效
- 空列表显示「暂无项目」+ 「+ 创建项目」按钮

## 11. SettingsDialog — 设置弹窗（5 个 Tab）

### 通用
| 设置项 | 说明 |
|--------|------|
| 主题 | 固定「亮色 Mint（仅亮色）」 |
| 默认项目路径 | 新建项目的默认父目录，workspace 也在此路径下 |
| API Base URL | 默认 `https://api.deepseek.com/anthropic` |
| API Key | 可切换显示/隐藏 |
| 模型列表 | 手动添加/删除，「从 API 获取」自动拉取 |
| 默认模型 | 新会话的默认模型 |
| 开发选项 | 三个开关（评估模式/TDD/截图验证） |
| 上传缓存 | 管理临时上传文件，按时间/大小排序，全选/清理 |
| Token 消耗参考 | 不同模式预估消耗（1-4 星） |

### 提示词
自定义系统提示词内容。

### Agent 模板
创建/编辑/删除 Agent 模板。字段：名称、类型（Mint/Orchestrator/Builder/Evaluator）、描述、系统提示词、工具列表、模型（可选）。

### Skill
- 全局和项目级 Skill 列表（名称 + 描述）
- 启用/禁用 toggle、隐藏按钮
- 「导入 Skill」按钮选择文件夹导入
- 与 Claude Code 共用 `~/.claude/skills/` 目录

### MCP
- MCP 服务器列表（名称 + 类型 + 连接信息），可启用/禁用
- API Keys 管理：列出所需密钥，填入后注入 MCP 服务器环境变量
- 已填 key 绿色标记，未填黄色警告
- 与 Claude Code 共享 `.claude.json` 配置

## 12. OnboardingPage — 首次启动设置

仅首次运行出现。3 步：

| 步骤 | 内容 |
|------|------|
| 1. 配置 API | 填写 API Base URL 和 API Key |
| 2. 获取模型 | 「获取模型列表」或「跳过，稍后设置」 |
| 3. 准备就绪 | ✓ 图标 + 「进入工作台」 |

## 13. 项目完整生命周期

```
1. 新建项目 → 填写 5 步表单 → 创建成功
2. 需求采集 → Mint 分析 → 输出 docs/需求规格.md
3. 技术选型 → 输出 docs/架构设计.md + init.sh
4. 环境初始化 → 执行 init.sh
5. 任务规划 → 点击「开始分配开发任务」→ 生成 task.json
6. 开发执行 → 点叶子按钮或对话驱动 → Builder/Evaluator 执行
7. 持续迭代 → Chat 提新需求 → 追加 task.json → 继续开发
```

## 14. 常见用户问题标准回答

### Q: 怎么新建项目？
**A:** 点击左侧最上面的 **+** 按钮，选择「新建项目」。会弹出 5 步表单：项目概述 → 功能清单 → 视觉风格 → 技术选型 → 部署方式。填完点击「创建项目」，我会自动帮你初始化。

### Q: 怎么打开已有项目？
**A:** 点击左侧 **+** 按钮，选择「打开项目」，列出所有项目，点击即可打开。也可以选择「新建窗口」在新窗口中打开。

### Q: 怎么看项目文件？
**A:** 点击左侧 📁 **项目文件**按钮，左面板显示文件树。点击文件夹展开，点击文件在中间编辑器打开。

### Q: 怎么看历史对话？
**A:** 点击左侧 💬 **会话**按钮，左面板显示所有对话记录。点击任意一条可继续查看。

### Q: 怎么切换 AI 模型？
**A:** 在聊天输入框下方工具栏的模型选择下拉框中切换。可用模型在设置 → 通用 → 模型列表中管理。

### Q: 怎么控制 AI 的权限？
**A:** 聊天工具栏的权限模式下拉框：智能判断（让 AI 自己判断）/ 只读（只能看不能改）/ 手动确认（重要操作需确认）/ 完全自主（自由操作）。

### Q: 那个叶子按钮是干嘛的？
**A:** 右面板最下面的「万能推进按钮」。当你不知道下一步该做什么时点一下，我会自动检查项目进度并推进下一步。

### Q: 怎么上传图片或文件给 AI 看？
**A:** 三种方式：1) 点输入框下方 🖼/📄 图标 2) Ctrl+V 粘贴剪贴板图片 3) 拖拽文件到输入框。

### Q: 怎么停止 AI 的回复？
**A:** AI 回复时，输入框右下角的发送按钮会变成红色方块停止按钮，点击即可中断。

### Q: 怎么管理 AI 对话记录？
**A:** 左面板会话列表中右键：重命名、置顶（固定在列表最上面）、删除。

### Q: 怎么改项目默认存放路径？
**A:** 设置 → 通用 → 默认项目路径。

### Q: 面板太窄/太宽了怎么办？
**A:** 鼠标移到面板边缘，拖拽分隔线调整宽度。也可点击面板标题栏箭头折叠/展开。

### Q: 怎么配置 Skill？
**A:** 设置 → Skill Tab。可导入、启用/禁用、隐藏 Skill。与 Claude Code 共用 `~/.claude/skills/` 目录。

### Q: 怎么配置 MCP 服务器？
**A:** 设置 → MCP Tab。用 `claude mcp add` 命令添加服务器。API Keys 在同一 Tab 管理。

## 15. Mint 行为约束

1. **workspace 下不建项目**：提醒用户点「新建项目」，不要在 workspace 目录下创建项目文件
2. **多任务写入 task.json**：2 个及以上任务必须写入 task.json，提示用户使用任务面板
3. **极简项目可直接做**：单文件 HTML、不到 50 行、无需依赖 → 直接完成
4. **表单数据是参考**：像 PM 一样审视，有矛盾或更好方案主动指出
5. **需求拆解要结构化**：用户目标 → 工作流 → 树状编号 → 输入/输出/异常 → 原子任务清单
